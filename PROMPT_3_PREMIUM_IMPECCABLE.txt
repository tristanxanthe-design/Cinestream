### DESIGN PHILOSOPHY
- RESTRAINT over excess — every element earns its place
- TYPOGRAPHY-DRIVEN — the type system carries the design, not decorations
- CINEMATIC — movie posters and backdrops are the visual centerpiece, UI gets out of the way
- MICRO-INTERACTIONS everywhere — hover states, focus states, transitions on everything
- AMBIENT — subtle background glows and noise texture create atmosphere without distraction
- NO DECORATION FOR DECORATION'S SAKE — if it doesn't serve the content, remove it

1. **Film grain noise overlay** — subtle cinematic texture:
```css
.noise::after {
  content: ''; position: fixed; inset: 0; z-index: 9999; pointer-events: none;

}
```

2. **Ambient glow system** — two floating gradient orbs behind content:
```css
.ambient-glow {
  position: fixed; border-radius: 50%; filter: blur(100px); pointer-events: none;
  animation: ambient-float 8s ease-in-out infinite;
}
@keyframes ambient-float { 0%,100%{transform:translate(0,0)} 50%{transform:translate(20px,-20px)} }
```
- Glow 1: 500px, top-center, rgba(229,9,20,0.04)
- Glow 2: 300px, bottom-right, rgba(167,139,250,0.04), animation-delay 4s

3. **Scroll progress indicator** — thin 2px red line at very top of viewport:
```css
.scroll-progress {
  position: fixed; top: 0; left: 0; height: 2px; z-index: 99999;
  background: #e50914; transform-origin: left;
  transition: none;
}
```
Update width via JS based on scroll percentage.

### MICRO-INTERACTIONS (12 required — this makes it feel $10k):

1. **Button press**: transform scale(0.97) on :active, instant. 0.15s ease transition on release.
2. **Card hover**: 0.3s transition, translateY(-4px), border color shift to rgba(255,255,255,0.1), shadow deepens.
3. **Link hover**: color transition 0.15s. Underline slides in from left (width 0% → 100%, using ::after pseudo-element).
4. **Focus rings**: box-shadow 0 0 0 2px rgba(229,9,20,0.3) on all interactive elements on :focus-visible.
5. **Page load stagger**: hero elements animate in sequence — chips (0s), title (0.1s), meta (0.15s), desc (0.2s), buttons (0.3s). Use framer-motion staggerChildren.
6. **Skeleton loaders**: animated pulse bg oscillating rgba(255,255,255,0.02) ↔ rgba(255,255,255,0.05) over 1.5s. Use for every async content block.
7. **Image reveal**: images start opacity 0, fade to 1 over 0.3s when loaded (onLoad handler).
8. **Scroll progress bar**: 2px red line at top of page grows with scroll position.
9. **Toast notifications**: glass panel slides in from bottom-right for "Added to watchlist", "Removed from watchlist". Auto-dismiss after 3s with fade out.
10. **Navbar scroll transition**: transparent → glass bg at 50px scroll, 0.3s transition.
11. **Carousel arrows**: opacity 0 by default, fade in on section hover, 0.2s transition.
12. **Rating tooltip**: hovering a rating badge shows a small tooltip "X/10 on TMDB" with glass bg, 0.15s fade in.

### RESPONSIVE
- Max content width: 1400px, mx-auto
- Below 1200px: hero single column, featured card stacks below
- Below 768px: simplified hover states, hamburger menu, no ambient glows (performance)
- Below 480px: full-width cards, compact spacing, no rank numbers

### ACCESSIBILITY
- All interactive elements keyboard-focusable
- :focus-visible rings (not :focus — only show for keyboard users)
- @media (prefers-reduced-motion: reduce) — disable all animations except opacity transitions
- Alt text on all next/image elements
- Semantic HTML: nav, main, section, article, h1-h3 hierarchy
- ARIA labels on icon-only buttons

### PERFORMANCE
- next/image for ALL images with width/height/priority
- Lazy load below-fold carousels
- CSS containment on card grid containers
- Debounce scroll event handlers with requestAnimationFrame
- No layout shift: explicit dimensions on all media

## PHASE 3: IMPECCABLE POLISH WORKFLOW

After the full build is working on localhost, run these commands IN ORDER.
Each step fixes a specific dimension. Do NOT skip steps.

### Step 1: Audit — find all issues
```
/audit
```
Fix every P0 (critical) and P1 (high) issue. These are blockers — accessibility violations, broken responsive layouts, performance problems. Log what was found and fixed.

### Step 2: Typeset — perfect the typography
```
/typeset
```
Let Impeccable review the full type system. It should respect our Playfair Display + Instrument Sans pairing. Fix any sizing inconsistencies, weight problems, line-height issues, or hierarchy gaps. Make sure the serif display headings and sans body text create clear visual contrast.

### Step 3: Arrange — fix layout and spacing
```
/arrange
```
Fix visual rhythm across all pages. Make sure glass panels breathe with proper padding. Check carousel spacing, section gaps, and the overall vertical flow. The premium feel comes from GENEROUS whitespace — don't cramp anything.

### Step 4: Colorize — review color usage
```
/colorize
```
Audit the color palette. Red (#e50914) should be used as a SURGICAL accent — watch buttons, active states, the scroll bar, badges. Not splashed everywhere. Most of the UI should be white text on near-black with glass borders. Check contrast ratios. Make sure rating colors (green/yellow/red) are accessible.

### Step 5: Animate — review all motion
```
/animate
```
Review every animation and micro-interaction. The 12 interactions listed above should ALL be implemented. Add any missing ones. Remove any that feel gratuitous or janky. Motion should feel INTENTIONAL — ease-out for entries, ease-in for exits. No bounce easing. No overshooting. Smooth and confident.

### Step 6: Harden — production-ready resilience
```
/harden
```
Add error handling for failed TMDB fetches (show graceful empty states). Handle text overflow on long movie titles (line-clamp). Handle missing poster images (fallback placeholder). Handle edge cases: empty watchlist, no search results, slow network. Add proper loading states everywhere.

### Step 7: Adapt — responsive perfection
```
/adapt
```
Review every page at 1920px, 1440px, 1200px, 1024px, 768px, 480px, 375px. Fix any breakpoint issues. Mobile should still feel premium — not a stripped-down version, but a thoughtfully adapted one. Touch targets should be minimum 44px.

### Step 8: Critique — UX design review
```
/critique
```
Get a full UX design review. Is the visual hierarchy clear? Does the eye flow naturally? Does the emotional tone feel cinematic and sophisticated? Is navigation intuitive? Fix any issues identified.

### Step 9: Distill — remove unnecessary complexity
```
/distill
```
Look for anything that doesn't serve the user. Unnecessary borders, redundant elements, cluttered sections. The premium feel comes from what you REMOVE, not what you add. If something can be simpler, simplify it.

### Step 10: Polish — final refinement
```
/polish
```
Final pass. Check every single page. Every hover state. Every transition. Every loading state. Every empty state. This is the ship-ready pass. Nothing should feel rough, unfinished, or inconsistent.

### Step 11: Final Audit — confirm clean
```
/audit
```
One last audit. P0 and P1 should be ZERO. P2 should be minimal. If any issues remain, fix them now.
