### BACKGROUND EFFECTS (apply to root layout)

1. **Perspective grid floor** — creates the Spline-like 3D ground plane:
```css
.grid-floor {
  position: fixed; inset: 0; pointer-events: none; z-index: 0;
  background-image:
    linear-gradient(rgba(255,255,255,0.02) 1px, transparent 1px),
    linear-gradient(90deg, rgba(255,255,255,0.02) 1px, transparent 1px);
  background-size: 80px 80px;
  transform: perspective(600px) rotateX(60deg);
  transform-origin: center top;
  opacity: 0.4;
}
```

2. **Floating orb glows** — 3 ambient blurred circles:
```css
.orb { position: fixed; border-radius: 50%; filter: blur(80px); pointer-events: none; animation: orb-float 8s ease-in-out infinite; }
@keyframes orb-float { 0%,100%{transform:translate(0,0)} 50%{transform:translate(20px,-20px)} }
```
- Orb 1: 300px, top-left, rgba(120,50,255,0.08)
- Orb 2: 250px, bottom-right, rgba(229,9,20,0.06), animation-delay 4s
- Orb 3: 200px, bottom-left, rgba(59,130,246,0.05), animation-delay 2s

3. **3D card tilt on mousemove** (apply to all media cards):
```javascript
function addTilt(card) {
  card.addEventListener('mousemove', (e) => {
    const rect = card.getBoundingClientRect();
    const x = (e.clientX - rect.left) / rect.width - 0.5;
    const y = (e.clientY - rect.top) / rect.height - 0.5;
    card.style.transform = `perspective(800px) rotateY(${x * 8}deg) rotateX(${-y * 8}deg) scale(1.02)`;
    card.style.transition = 'transform 0.1s ease-out';
  });
  card.addEventListener('mouseleave', () => {
    card.style.transform = 'perspective(800px) rotateY(0) rotateX(0) scale(1)';
    card.style.transition = 'transform 0.4s ease-out';
  });
}
```
Wrap in requestAnimationFrame for performance. Use will-change: transform.

4. **Scroll-triggered animations** — fade+slide every section on viewport entry:
```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) entry.target.classList.add('visible');
  });
}, { threshold: 0.1 });
```
CSS: opacity 0 → 1, translateY 30px → 0, 0.6s ease-out. Stagger children +0.05s each.

## PHASE 3: IMPECCABLE POLISH WORKFLOW

After the build is complete and working on localhost, run these commands IN ORDER:

### Step 1: Audit
```
/audit
```
Fix every P0 and P1 issue it finds. This catches accessibility, performance, responsive problems.

### Step 2: Typeset
```
/typeset
```
Let it fix any typography inconsistencies — font sizing, hierarchy, weight, spacing, line-height. Make sure it respects our Syne + Space Grotesk system.

### Step 3: Arrange
```
/arrange
```
Fix layout spacing, visual rhythm, alignment. Make sure the glassmorphism panels breathe properly.

### Step 4: Colorize
```
/colorize
```
Review the color palette. Make sure purple and red accents are used intentionally, not splashed everywhere. Ensure contrast ratios are accessible.

### Step 5: Animate
```
/animate
```
Review all animations. Add purposeful micro-interactions where missing — hover states, focus states, page transitions. Remove any that feel gratuitous.

### Step 6: Overdrive
```
/overdrive
```
This is the Spline magic — push for technically extraordinary effects:
- Scroll-driven reveals on sections
- Spring physics on card interactions
- Parallax depth on the hero area
- Ambient particle effects if tasteful
- Smooth scroll with momentum

### Step 7: Critique
```
/critique
```
Get a UX design review. Fix any hierarchy, clarity, or emotional resonance issues.

### Step 8: Polish
```
/polish
```
Final pass. Every pixel, every transition, every hover state. Ship-ready.

### Step 9: Final Audit
```
/audit
```
Run one more audit to confirm all fixes are clean. P0 and P1 should be zero.
